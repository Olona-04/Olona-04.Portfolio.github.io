# Reflection on Coding in Markdown

## Situation
During the development of my digital portfolio, I was required to create and code my CV using Markdown language. At first, I was not very familiar with Markdown and had only basic knowledge about formatting text. I realized that this task would require me to learn a new skill and apply it professionally. I also needed to ensure that my CV was well-structured, clear, and visually appealing when viewed on GitHub. This was important because my digital portfolio represents my professional identity and work readiness.

In addition, I understood that Markdown is widely used by developers and IT professionals, so learning this skill would be beneficial for my future career. I wanted to make sure that my CV looked professional and easy to read, while also demonstrating my technical abilities.

## Task
My task was to learn Markdown and use it to create a professional CV. I needed to structure my document properly using headings, bullet points, bold text, links, and spacing. I also needed to make sure that my CV included important sections such as:

- Personal Information  
- Contact Details  
- Education  
- Skills  
- Projects  
- Work Experience  
- References  

I also needed to ensure that the document displayed correctly on GitHub and that it looked clean and professional.

## Action
To complete this task, I started by researching Markdown syntax and learning how it works. I practiced using headings, bullet points, bold text, and spacing. I also explored how to add links and organize content properly. I reviewed examples of professional CVs written in Markdown and used them as a guide.

I also tested my Markdown file multiple times on GitHub to ensure that everything displayed correctly. I made several improvements to the layout and formatting to make the CV more readable and professional. I ensured consistency in spacing, headings, and formatting throughout the document.

During this process, I also learned:

- How to structure professional documents  
- How to format content using Markdown  
- How to create clean and readable layouts  
- How to improve readability and presentation  
- How to use GitHub effectively

### What Was Easy
Some parts of Markdown were easy to learn, such as creating headings, bullet points, and bold text. Once I understood the basic syntax, it became easier to organize my CV. Adding sections and formatting text was also simple after some practice.

### What Was Difficult
Some parts were challenging, especially when formatting spacing and ensuring everything displayed correctly on GitHub. At times, I struggled with aligning content and making the document look professional. Learning how to add links and maintain consistent formatting also required extra effort. However, with practice and testing, I was able to overcome these challenges.

## Result
As a result, I successfully created a professional CV using Markdown. My document became well-structured, easy to read, and visually appealing. I gained confidence in using Markdown and improved my technical skills. I also developed a better understanding of how developers present documentation professionally.

This experience helped me improve my attention to detail, formatting skills, and overall professionalism. I am now more confident in using Markdown for future projects and documentation.
